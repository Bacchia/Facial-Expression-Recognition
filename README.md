# Facial Expression Recognition (FER2013) — Pipeline Optimization

სახის ემოციების ამოცნობა ღრმა სწავლების (Deep Learning) მეთოდებით, Kaggle-ის კონკურსის
[*Challenges in Representation Learning: Facial Expression Recognition Challenge*](https://www.kaggle.com/c/challenges-in-representation-learning-facial-expression-recognition-challenge)
მიხედვით.

ეს რეპოზიტორია დეტალურად აღწერს ჩემს მიერ განვლილ **23-ეტაპიან საექსპერიმენტო გზას** — როგორ ავქაჩე
საწყისი ~25%-იანი საბაზისო სიზუსტიდან (baseline) მაქსიმალურ **66.34%**-მდე, ნაბიჯ-ნაბიჯ, ერთი
არქიტექტურული გადაწყვეტილებიდან მეორეში გადასვლით.

ექსპერიმენტების სრული, ინტერაქტიული მართვის პანელი და გრაფიკები ხელმისაწვდომია Weights & Biases-ზე:
🔗 **[FER2013-Project @ W&B Dashboard](https://wandb.ai/bacchia-free-university-of-tbilisi-/FER2013-Project)**

## 1. ამოცანის არსი და მთავარი სირთულეები

FER2013 ემოციების კლასიფიკაციის კლასიკური ბენჩმარკია:

- **მონაცემები:** 48×48 ზომის შავ-თეთრი (grayscale, 1 არხი) სურათები.
- **7 სამიზნე კლასი:** `0=Angry`, `1=Disgust`, `2=Fear`, `3=Happy`, `4=Sad`, `5=Surprise`, `6=Neutral`.
- **ფორმატი:** CSV ფაილი, სადაც `pixels` სვეტში წერია გამოტოვებით (space) გამოყოფილი 2304 (= 48×48) რიცხვი.

ამოცანა — `test.csv`-ის თითოეული სურათისთვის ემოციის პროგნოზი და `submission.csv`-ის გენერაცია.

### რატომ არის ეს ამოცანა რთული?

**ხმაურიანი ლეიბლები (Noisy Labels).** ბევრ სურათზე თავად ადამიანისთვისაც კი ძნელი სათქმელია,
კონკრეტულად რა ემოციაა გამოსახული — `Fear`, `Sad` და `Neutral` ხშირად ერთმანეთს ემსგავსება.
სწორედ ამიტომ ადამიანის საშუალო სიზუსტეც ამ მონაცემებზე მხოლოდ ~65%-ის ფარგლებშია. ეს ფაქტი
მნიშვნელოვანია: 66%-იანი მოდელი უკვე ადამიანის დონეზე მუშაობს.

**კლასების დისბალანსი (Data Imbalance).** ემოციები უკიდურესად არათანაბრადაა განაწილებული.
`Disgust` ისეთი იშვიათია, რომ მისი გამოთვლილი class weight ≈ **9.4**-ია, მაშინ როცა `Happy` და
`Neutral` თითქმის ყველგან გვხვდება (წონა ~1-ის ფარგლებში). თუ ამას ანგარიში არ გავუწიეთ, მოდელი
უბრალოდ ხშირ კლასებს „დაიზეპირებს" და იშვიათებს სრულად დაკარგავს.

გარემო. ეს იმას ნიშნავს, რომ ნებისმიერი ექსპერიმენტი აღიწერება **მხოლოდ ერთი YAML ფაილით**, კოდი

## 3. მონაცემების მომზადება (Data Pipeline)

### 3.1. სტრატიფიცირებული გაყოფა (Stratified Split)

რადგან მონაცემები არაბალანსირებულია, ჩვეულებრივი შემთხვევითი გაყოფით ვრისკავთ, რომ იშვიათი
კლასები (`Disgust`) საერთოდ ვერ მოხვდეს ვალიდაციის ნაწილში — და მაშინ ვალიდაციის სიზუსტე უბრალოდ
მოგვატყუებს. ამის გამოსასწორებლად გამოვიყენე **Stratified Split** (80/20 პროპორციით), რომელმაც
უზრუნველყო კლასების იდენტური განაწილება როგორც Train, ისე Validation სეტში.

გაყოფა ფიქსირებულია `random_state=42`-ით, ანუ ყველა 23 ექსპერიმენტი **ზუსტად ერთსა და იმავე**
train/val ნაწილზე შევადარე — სხვაგვარად შედეგების შედარება არასამართლიანი იქნებოდა.

### 3.2. Data Augmentation

ტრენინგზე ვიყენებ ორ ტრანსფორმაციას:

- `RandomHorizontalFlip(p=0.5)` — რადგან სახე ჰორიზონტალურად სიმეტრიულია, ფლიპი სრულიად ბუნებრივი
  და უსაფრთხო augmentation-ია.
- `RandomRotation(degrees=10)` — თავის რეალური, მცირე გადახრის სიმულაცია.

აგრესიულ ტრანსფორმაციებს (ვერტიკალური ფლიპი ან დიდი როტაცია) **განზრახ** ავარიდე თავი: მათ შეუძლიათ
ემოციის აზრი მთლიანად შეცვალონ — მაგალითად, ვერტიკალურმა ფლიპმა ქვემოთ დაწეული ტუჩის კუთხეები
ზემოთ რომ ასწიოს, სევდა „ღიმილად" გადაიქცევა და ლეიბლი მცდარი გახდება.

ვალიდაციაზე augmentation საერთოდ არ მაქვს — შეფასება უნდა იყოს დეტერმინისტული და სუფთა.

## 4. მოდელების ევოლუცია და საინჟინრო გადაწყვეტილებები

23 გაშვების (run) განმავლობაში მოდელი მარტივიდან რთულისკენ განვავითარე. ეს განზრახ გავაკეთე ასე —
რომ თითოეული ახალი კომპონენტის **რეალური წვლილი** ცხადად დამენახა, და არა ბრმად დამეგროვებინა
სირთულეები.

### 4.1. Linear Baseline (~35% Val Accuracy)

უბრალო ერთი ხაზოვანი ფენა, `Linear(2304 → 7)`. ეს მოდელი ვერაფერს „ხედავს" სივრცულად — უბრალოდ
პიქსელების შეწონილ ჯამს იღებს. გამოვიყენე მხოლოდ „იატაკის" დასაფიქსირებლად: თუ რომელიმე რთული
მოდელი ამაზე უარესი იქნებოდა, ე.ი. სადღაც კოდში ლოგიკური შეცდომა მქონდა. 35% (random guess ~14%-ის
წინააღმდეგ) დამიდასტურა, რომ pipeline მუშაობს, მაგრამ ხაზოვნად ამოცანა ვერ გადაიჭრება.

### 4.2. SimpleCNN (~60–62% Val Accuracy)

4 კონვოლუციური ბლოკი არხების ზრდით (`32→64→128→256`), თითოეულში `Conv → BatchNorm → ReLU → MaxPool`,
ბოლოს `Linear(256·3·3 → 512) → Dropout(0.4) → Linear(512 → 7)`.

- **BatchNorm** — სტაბილური და სწრაფი ტრენინგისთვის.
- **MaxPool ×4** — სივრცული შეკუმშვა `48→24→12→6→3`.
- **Dropout(0.4)** — overfitting-ის შესაკავებლად.

ეს იყო ყველაზე დიდი ერთჯერადი ნახტომი (35% → 62%) — დაგვიდასტურა, რომ კონვოლუციური (სივრცული)
ინდუქციური bias ამ ამოცანისთვის აუცილებელია. მოდელმა მაინც სწრაფად დაიწყო ზედმეტი მორგება (overfitting).

### 4.3. კოლაფსი და გაკვეთილი: WeakResNet (25.13%)

ერთ-ერთ ექსპერიმენტში (`resnet_boosted_v2`) ქსელი სრულად **კოლაფსირდა** და გაიჭედა 25.13%-ზე ყველა
ეპოქის განმავლობაში. ეს ნიშნავდა, რომ მოდელმა უბრალოდ ერთ, ყველაზე ხშირ კლასზე დაიწყო მუდმივი
პროგნოზი — ის აღარაფერს სწავლობდა.

- **მიზეზი:** არასტაბილური არქიტექტურისთვის ძალიან მაღალი Learning Rate (`0.0035`) ავირჩიე —
  გრადიენტები „აფეთქდა" და მოდელი დეგენერაციულ მინიმუმში ჩავარდა.
- **გამოსავალი:** ეს წარუმატებელი run **განზრახ შევინახე W&B-ზე** — ის ნათლად აჩვენებს, რა არ უნდა
  გავაკეთო. ამის შემდეგ გადავედი გაცილებით დაბალ საწყის LR-ზე (`0.0004`) და უფრო სტაბილურ
  სტრუქტურაზე (`SolidResNet`), რომელმაც ~61% მოგვცა.

### 4.4. UltimateResNet და Pre-Activation ბლოკები (64.66% Base → 66.34% TTA) 🏆

საბოლოო ჩემპიონ არქიტექტურად ავირჩიე **Pre-Activation ResNet** (`BN → ReLU → Conv` თანმიმდევრობით,
He et al. 2016), 4 ნარჩენი ბლოკით `64→128→256→512`. ამ სტრუქტურაში გრადიენტი ბევრად უფრო სუფთად,
დანაკარგების გარეშე მიედინება skip connection-ებში, რაც ღრმა ქსელის ტრენინგს ბევრად სტაბილურს ხდის
(სწორედ აქ წააგო `WeakResNet`-მა).

დამატებითი გადაწყვეტილებები ამ მოდელისთვის:

- **AdamW** ოპტიმიზატორი (და არა Adam) — სწორი, decoupled weight decay, რაც გაცილებით ჯანსაღ
  რეგულარიზაციას იძლევა.
- **Weight Decay (0.01)** და **Dropout(0.5)** — ძლიერი რეგულარიზაცია, რადგან მოდელი train-ს ძალიან
  კარგად ფიტავს (train acc ~88%, val ~65% — ცხადი overfitting-ის ნიშანი, რომელსაც სწორედ ეს
  ტექნიკები აკონტროლებენ).
- **Label Smoothing (0.1)** — აიძულა მოდელი, რომ ხმაურიან ლეიბლებზე ზედმეტად თავდაჯერებული არ
  ყოფილიყო; ცდომილ ლეიბლზე „დარწმუნება" ისჯება.
- **CosineAnnealingLR** — learning rate cosine მრუდით თანდათან ეცემა, რაც მოდელს ტრენინგის ბოლოს
  უფრო ღრმა, სტაბილურ მინიმუმში დაჯდომაში ეხმარება.

### 4.5. Hybrid ViT (~50% Val Accuracy)

ცალკე ვცადე ჰიბრიდული არქიტექტურა: CNN backbone იღებს ლოკალურ ფიჩერებს, შემდეგ ფიჩერ-რუკა იშლება
„ტოკენებად", ემატება პოზიციური embedding და გადის Multi-Head Self-Attention ბლოკებში. სუფთა ViT-ის
ნაცვლად ჰიბრიდი ავირჩიე იმიტომ, რომ transformer-ს ბევრი მონაცემი სჭირდება, FER2013 კი პატარაა.
შედეგი ~50%-ზე გაჩერდა — ResNet-ს ჩამორჩა. ეს კარგი დადასტურებაა, რომ ამ მასშტაბზე attention-ის
უპირატესობა არ ვლინდება, თუმცა ექსპერიმენტი თავად ღირებული იყო.

## 5. საბოლოო კონფიგურაცია (`resnet_ultimate.yaml`) და როგორ მივედი მასთან

თითოეული ექსპერიმენტი ცალკე YAML ფაილით აღიწერება, ხოლო `resnet_ultimate.yaml` არის ის საბოლოო
კონფიგი, რომელმაც ჩემპიონ შედეგამდე მიმიყვანა:

```yaml
experiment_name: "ultimate_resnet_mixup_v12"
model_name: "UltimateResNet"
learning_rate: 0.0012
weight_decay: 0.01
label_smoothing: 0.10
epochs: 35
batch_size: 64
test_size: 0.2
random_state: 42
```

ეს რიცხვები ერთბაშად არ დაბადებულა — თითოეულ მათგანამდე წინა ექსპერიმენტებიდან მივედი, ხელით,
ერთ პარამეტრს რომ ვცვლიდი და W&B-ზე ვაკვირდებოდი შედეგს:

- **`learning_rate: 0.0012`** — `WeakResNet`-ის კოლაფსმა (LR `0.0035`) მკაფიოდ მაჩვენა, რომ მაღალი
  LR ამ ღრმა ქსელს „კლავს". `SolidResNet`-ზე ძალიან ფრთხილი `0.0004` ავიღე და სტაბილური, მაგრამ ნელი
  ტრენინგი მივიღე. `0.0012` ამ ორ უკიდურესობას შორის ოქროს შუალედია — სწრაფი, მაგრამ უსაფრთხო
  (განსაკუთრებით AdamW-სა და cosine scheduler-თან ერთად).
- **`weight_decay: 0.01`** — `SimpleCNN`-ისა და `SolidResNet`-ის overfitting-მა (train ≫ val)
  მაიძულა, რეგულარიზაცია გამეძლიერებინა. სტანდარტული `1e-4`-დან `0.01`-მდე აწევამ train/val
  სხვაობა შესამჩნევად შეამცირა.
- **`label_smoothing: 0.10`** — მონაცემების ხმაურის გამო (ლეიბლები ხშირად საეჭვოა) მოდელს არ უნდა
  მივცე უფლება, ცდომილ ლეიბლზე 100%-ით „დარწმუნდეს". 0.10 აღმოჩნდა ის ზღვარი, რომლის ზემოთაც
  მოდელი უკვე ზედმეტად „გაუბედავი" ხდებოდა.
- **`epochs: 35`** — წინა ResNet-ებზე ვალიდაცია ~25 ეპოქის შემდეგ კიდევ ნელა იზრდებოდა; ვინაიდან
  cosine scheduler-ს მეტი ეპოქა სჭირდება სრულად „გასაცივებლად", 35-მდე ავწიე.
- **`batch_size: 64`** — `32`-ით ტრენინგი ხმაურიანი იყო, `128`-ით კი ვალიდაცია ოდნავ ეცემოდა; `64`
  საუკეთესო ბალანსი აღმოჩნდა.
- **`test_size: 0.2`, `random_state: 42`** — ფიქსირებული, ყველა ექსპერიმენტში უცვლელი, რომ
  შედარება სამართლიანი ყოფილიყო.

მოკლედ, საბოლოო YAML არ არის „გამოცნობილი" — ის წინა ცდების (baseline → CNN → კოლაფსი → SolidResNet)
პირდაპირი დასკვნაა.

## 6. გარდამტეხი მომენტი: TTA vs MixUp

ექსპერიმენტების ბოლო ფაზაში ორი ძლიერი რეგულარიზაციის ტექნიკა დაუპირისპირდა ერთმანეთს — და
საინტერესოა, რომ უფრო რთული მათგანი არ აღმოჩნდა საუკეთესო.

### 6.1. Run v11 (`ultimate_resnet_tta_v11`) — პროექტის პიკი 🏆

ამ მოდელმა სტანდარტულ ვალიდაციაზე **64.66%** აჩვენა. თუმცა ტრენინგის დასრულებისთანავე ჩაირთო
სპეციალური **Test-Time Augmentation** შეფასების ციკლი: თითოეული სურათი მოდელს მიეწოდა 3 ვარიაციით —

1. ორიგინალი სურათი,
2. ჰორიზონტალურად შემოტრიალებული (flipped),
3. 2 პიქსელით გვერდზე გაჩოჩებული (shifted).

მოდელმა სამივე ვერსიის ალბათობები გაააშუალა და საბოლოო პასუხი ამ „კონსენსუსით" დაადგინა. ამან
წაშალა მცირე სივრცული ცდომილებები (ერთგვარი იაფი ансамბლი ერთი მოდელით) და სიზუსტე მყისიერად
**66.34%**-მდე ასწია.

### 6.2. Run v12 (`ultimate_resnet_mixup_v12`) — რატომ არ ავირჩიე?

ამ გაშვებაში დამატებით ჩავრთე **MixUp** რეგულარიზაცია — ბაჩში სურათების ერთმანეთში არევა
(მაგ. 70% `Angry` + 30% `Sad`, ლეიბლების იმავე პროპორციით შერევით). საბაზისო სიზუსტე ოდნავ მაღალი
გამოვიდა, **65.06%**, მაგრამ MixUp-მა ქსელის გადაწყვეტილების საზღვრები იმდენად გლუვი გახადა, რომ
TTA-ს დამატებამ დიდი ეფექტი ვეღარ მოიტანა — მოდელი **65.53%**-ზე გაჩერდა.

ანუ ორმა „გამარბილებელმა" ტექნიკამ ერთმანეთს ეფექტი დაუკარგა. შესაბამისად, საბოლოო Champion Run-ად
**v11** მივიჩნიე (W&B run ID: `dv4e2sz0`).

| Run | აღწერა | Base Val Acc | + TTA |
|-----|--------|--------------|-------|
| **v11** `ultimate_resnet_tta_v11` 🏆 | Pre-Act ResNet + TTA | 64.66% | **66.34%** |
| v12 `ultimate_resnet_mixup_v12` | + MixUp | 65.06% | 65.53% |

## 9. ძირითადი დასკვნები

- **არაბალანსირებული მონაცემები** → Stratified Split + class weights + label smoothing.
- **ხმაურიანი ლეიბლები** → label smoothing (და, საჭიროებისამებრ, MixUp).
- **Overfitting** → Dropout, weight decay, augmentation.
- **ღრმა ქსელის სტაბილურობა** → Pre-Activation residual ბლოკები + დაბალი LR.
- **შეფასების სტაბილურობა** → TTA — სწორედ მან მოიტანა საბოლოო +1.68% და 66.34%-იანი პიკი.

ყველაზე დიდი გაკვეთილი: ხანდახან მარტივი ხრიკი (TTA) უფრო ეფექტურია, ვიდრე რთული ტექნიკის (MixUp)
ზედ დადება — და სწორედ ექსპერიმენტების სისტემურმა, W&B-ზე დალოგვამ მომცა ამის ნათლად დანახვის
საშუალება.
